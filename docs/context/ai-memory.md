- 2026-05-27：正式系统面向重写；旧原型在需求提取后废弃，不构成实现约束。
- 2026-05-27：首版采用 Python 模块化 MVC 单体，聚焦瞭望到问数闭环。
- 2026-05-27：数据库和 API 需要按目标系统正式设计，旧材料仅抽取需求。
- 2026-05-27：正式变更采用 Conventional Commits，经工作分支交付并由项目负责人 Code Review 后合并，AI 不直接提交或合并主分支。

- 2026-05-27：Python 工程统一使用 `uv` 管理依赖、虚拟环境和锁文件；依赖变更需同步提交 `pyproject.toml` 与 `uv.lock`，项目命令通过 `uv run ...` 执行。
- 2026-05-27：Phase 1 C 管理端页面调用正式 `/api/v1` 接口，不在页面层绕过权限、CSRF 或后端业务规则。
