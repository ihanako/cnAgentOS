# 项目 AI 记忆

本文件用于记录对后续开发长期有价值、且已由实现或用户确认的简短事实。它不是任务流水账，也不能替代正式需求、数据库或 API 契约。

## 已确认长期事实

- 2026-05-27：正式系统面向重写；旧原型在需求提取后废弃，不构成实现约束。
- 2026-05-27：首版采用 Python 模块化 MVC 单体，聚焦瞭望到问数闭环。
- 2026-05-27：数据库和 API 需要按目标系统正式设计，旧材料仅抽取需求。
- 2026-05-27：正式变更采用 Conventional Commits，经工作分支交付并由项目负责人 Code Review 后合并，AI 不直接提交或合并主分支。
- 2026-05-27：三人协作按 MVP 验收文档和 Phase 开发计划推进，每个工作包应有唯一负责人并经评审合并。
- 2026-05-27：Python 工程统一使用 `uv` 管理依赖、虚拟环境和锁文件；依赖变更需同步提交 `pyproject.toml` 与 `uv.lock`，项目命令通过 `uv run ...` 执行。
- 2026-05-27：Phase 1 C 管理端页面调用正式 `/api/v1` 接口，不在页面层绕过权限、CSRF 或后端业务规则。
- 2026-05-27：Phase 1 C 用户、角色和功能导航页面应通过现有管理 API 完成编辑、启停、删除、密码重置和角色/权限选择。
- 2026-05-27：认证请求可以聚合页面初始化读取，但 RBAC 权限不得固化到登录会话；角色停用或权限移除必须在后续受保护请求立即生效。
- 2026-05-28：仓库采用单仓前后端分离开发边界：`backend/` 承载 FastAPI/uv/Alembic/pytest，`frontend/` 承载 Vite Vue TypeScript/pnpm/Element Plus，`docs/` 仍在根目录作为共享契约；后端只提供 API，不托管前端页面或 `frontend/dist`。
- 2026-05-28：Phase 2 C 数据源、采集任务和数据仓库治理页面按 `docs/api/watch-and-data.md` 调用正式 API，不在前端内置 mock 数据。
- 2026-05-29：Phase 2 已完成智能瞭望与数据仓库收口；后端 `watch_and_data` API、采集安全、内容治理、Vue 管理端页面和系统导航均已通过自动化验证。
- 2026-05-29：修改 `SYSTEM_FUNCTIONS` 时每一项必须保持 7 元组格式；重复或旧格式导航项会导致 bootstrap 解包失败并阻断所有依赖初始化的后端测试。
- 2026-05-29：Phase 3 B/C 实现问数 API 和页面时必须复用 `services/qa_security.py`，通过查询级所有权过滤保护会话/回答/引用，并让他人记录和不存在记录统一返回 `404`。
