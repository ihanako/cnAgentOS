# 待解决问题与风险

## 实现前必须确认

| 事项 | 当前判断 | 处理时机 |
| --- | --- | --- |
| 前端实现方式 | 需要管理端与用户问数界面；可重写原型 | UI 实施计划中决定 |
| 内容检索方式 | 首版必须从可用数据取得依据；具体全文/向量策略待定 | 问数实施前决定 |
| Git 仓库与主分支保护 | Git 仓库和 `main` 已存在；远端主分支保护与评审门禁是否落地待确认 | 首批功能合并前验证分支保护与评审流程 |
| 登录接口防暴力保护 | 已统一认证失败响应，但尚未实现失败频率限制、失败审计或等效的口令猜测防护 | 生产部署与安全验收前确定策略并实现 |

## 高风险要求

| 风险 | 必须采取的控制 |
| --- | --- |
| 模型或来源凭据泄露 | 加密存储、脱敏响应、不进入日志与审计详情 |
| 采集造成 SSRF | 域名白名单、地址解析校验、重定向复查、超时与大小限制 |
| 权限仅在页面控制 | 所有管理 API 执行服务端权限判定 |
| AI 使用不可信采集内容 | 将内容作为资料处理，防止内容注入改变系统指令 |
| 旧原型秘密或演示配置被复用 | 重写时不复制任何原型凭据；若曾在外部使用过演示秘密，应予以轮换 |
